ğŸ“˜ What are Docker Services?

Services are the definition of how containers behave in a Docker Swarm cluster. A service includes the image to use and instructions on how to run containers (e.g., ports, environment, replicas).

ğŸ”§ Create a Simple Service

Start by initializing Swarm mode:

docker swarm init

Create a service:

docker service create --name web -p 8080:80 nginx

ğŸ“‹ Service Commands

  • docker service ls - List all services
  • docker service ps web - List tasks (containers) of the "web" service
  • docker service inspect web - Inspect a service
  • docker service scale web=3 - Scale service to 3 replicas
  • docker service update --image nginx:alpine web - Update service image
  • docker service rm web - Remove the service

Imagine you have a web app and want to load-balance it across multiple nodes. You can deploy it as a service with multiple replicas, allowing Docker to manage placement and updates automatically.

docker service create --name myapp -p 5000:5000 --replicas 5 myimage
