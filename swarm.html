⚡ Docker Swarm

Orchestration and Clustering with Swarm Mode

🔍 What is Docker Swarm?

Docker Swarm is Docker's native clustering and orchestration tool. It allows multiple Docker engines (hosts) to work together in a cluster, where services can be deployed, managed, and scaled across nodes.

🚀 Initialize Swarm Mode

docker swarm init

This initializes the current Docker host as a Swarm Manager.

🔗 Join a Worker Node

docker swarm join --token <token> <manager-ip>:2377

This command (generated after swarm init) lets other machines join as workers.

📜 List Nodes

docker node ls

Run this on the manager to see all nodes in the swarm cluster.

🛠️ Create a Service

docker service create --name web -p 8080:80 nginx

This deploys a replicated nginx container across the cluster.

📈 Scale a Service

docker service scale web=3

Increases the number of running instances (replicas) of the web service to 3.

📋 Inspect Services & Tasks

docker service ls

# Inspect a specific service
docker service inspect web --pretty

# View running containers of a service
docker service ps web

🧼 Remove Services & Leave Swarm

docker service rm web

# Leave swarm (on a worker)
docker swarm leave

# Force leave (on a manager)
docker swarm leave --force

📦 Real-World Use Case Example

Here’s a real-world example using Docker Swarm to deploy a web application:

# Initialize swarm on manager
$ docker swarm init

# Deploy service
$ docker service create --name frontend -p 80:80 myapp

# Scale up
$ docker service scale frontend=4

# View tasks
$ docker service ps frontend

# Update service image
$ docker service update --image myapp:v2 frontend
